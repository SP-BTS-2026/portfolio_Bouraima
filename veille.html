🤖 Veille n°1 : Les Agents IA Autonomes

Sujet : L'évolution des LLM vers les Agents IA Autonomes.

Ma veille porte sur la transition entre les IA conversationnelles (Chatbots) et les agents capables d'exécuter des tâches complexes de manière autonome.

Qu'est-ce qu'un Agent IA ?

Contrairement à un simple ChatGPT, un agent possède :

  • La Planification : Capacité à décomposer un objectif en étapes.
  • L'usage d'outils : Capacité à utiliser le web, du code Python ou des APIs.
  • La Mémoire : Utilisation de bases de données vectorielles (RAG).

Outils et Frameworks suivis

  • LangChain / LangGraph : Pour créer des cycles de raisonnement.
  • CrewAI : Pour faire collaborer plusieurs agents entre eux.
  • OpenAI Assistants API : L'implémentation native des agents.

Méthodologie de veille

Pour rester à jour, j'utilise : Feedly (RSS), Newsletter "The Rundown AI", YouTube et X.

Impact sur le métier de développeur SLAM

Le passage aux agents IA transforme notre rôle : nous devenons des "architectes d'agents".

🔐 Veille n°2 : La Sécurité des APIs REST

Sujet : Les bonnes pratiques et les menaces liées à la sécurisation des APIs REST.

Comprendre comment sécuriser ces points d'entrée, souvent ciblés par les attaquants.

Menaces principales

  • Injection SQL : Requêtes malveillantes.
  • Broken Authentication : Tokens mal gérés.
  • Exposition de données : Retours trop verbeux.

Bonnes pratiques suivies

  • JWT (JSON Web Token)
  • HTTPS obligatoire
  • Rate Limiting
  • Validation des entrées

Impact sur le métier de développeur SLAM

Adoption des réflexes de "Security by Design" dans chaque projet.
